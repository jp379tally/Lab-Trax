---
name: Draft-invoice preview must reuse syncInvoiceFromRestorations grouping
description: Which code path actually produces a new case's draft invoice, so any "preview" matches it.
---

A new case's **draft invoice is generated by `syncInvoiceFromRestorations`**
(in `artifacts/api-server/src/lib/invoice-sync.ts`), which builds its lines via
`buildBridgeAwareLineItems`. POST /cases calls `syncInvoiceFromRestorations`
after inserting restorations — it does NOT build the persisted invoice from the
inline, ungrouped line builder that also lives in the create path.

**Why:** building anything that must match the real draft invoice (e.g. the
read-only invoice preview in the Desktop drop-zone Rx panel) by mirroring the
POST /cases *inline* builder would silently diverge — that builder is ungrouped
and does not collapse same-material groups or pontic bridge spans, while the
actual stored invoice does.

**How to apply:** reuse the grouping helpers (now extracted into the shared pure
module `artifacts/api-server/src/lib/invoice-line-grouping.ts` — `buildBridgeAwareLineItems`,
`buildGroupedSyncItems`, bridge/connector helpers). Resolve pricing with the same
batch `resolveAllPricesForContext` + per-restoration `resolveServerPriceWithSource`
fallback that POST /cases uses, then group with `buildBridgeAwareLineItems`. Do
not reimplement grouping; do not copy the inline ungrouped builder.

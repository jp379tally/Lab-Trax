Yes — this needs to be tightened up in the edit-case flow.

Right now you have two bugs:

1. **Admin can edit fields, but there’s no save action**
2. **After saving, there’s no follow-up to reprint the updated case label**

And the rule should be:

* only **admin user** can edit case info
* when admin taps **Save Changes**, persist the edits
* after successful save, ask:
  **“Do you want to reprint the case label?”**
* if **No** → done
* if **Yes** → open the label/card print flow

## Target behavior

### Non-admin user

* can view case info
* cannot edit case info

### Admin user

* can open **Edit Case Info**
* can change fields
* sees a **Save Changes** button
* after save:

  * **No** → close and refresh
  * **Yes** → print updated case label

# Code pattern to use

## 1. Gate editing by admin role

Use a boolean like:

```ts
const canEditCaseInfo =
  currentMembership?.role === "admin" || currentMembership?.role === "owner";
```

Then only show the edit button if allowed:

```tsx
{canEditCaseInfo && (
  <Pressable onPress={() => setEditCaseModalVisible(true)} style={styles.editBtn}>
    <Ionicons name="create-outline" size={18} color="#fff" />
    <Text style={styles.editBtnText}>Edit Case Info</Text>
  </Pressable>
)}
```

If only true admin, not owner, then use exactly:

```ts
const canEditCaseInfo = currentMembership?.role === "admin";
```

## 2. Use draft state inside the edit modal

Do not bind form fields directly to the live case object.

```ts
const [editCaseModalVisible, setEditCaseModalVisible] = useState(false);
const [isSavingCaseInfo, setIsSavingCaseInfo] = useState(false);

const [caseInfoDraft, setCaseInfoDraft] = useState({
  doctorName: "",
  patientName: "",
  dueDate: "",
  shade: "",
  notes: "",

```

When modal opens:

```ts
function openEditCaseInfo() {

    dueDate: selectedCase?.dueDate || "",
    shade: selectedCase?.shade || "",
    notes: selectedCase?.notes || "",

  setEditCaseModalVisible(true);
}
```

## 4. Save to backend, then refresh case, then ask about reprint

Use a handler like this:

```ts
async function handleSaveCaseInfo() {
  if (!selectedCase?.id) return;
  if (!canEditCaseInfo) {
    Alert.alert("Permission Denied", "Only admin users can edit case info.");
    return;
  }

  setIsSavingCaseInfo(true);

  try {
    const payload = {
      doctorName: caseInfoDraft.doctorName.trim(),
      patientName: caseInfoDraft.patientName.trim(),
      dueDate: caseInfoDraft.dueDate,
      shade: caseInfoDraft.shade.trim(),
      notes: caseInfoDraft.notes.trim(),
    };

    const res = await api.patch(`/cases/${selectedCase.id}`, payload);

    const updatedCase = res.data?.case || res.data;

    setSelectedCase(updatedCase);
    setCases((prev) =>
      prev.map((c) => (c.id === updatedCase.id ? updatedCase : c))
    );

    setEditCaseModalVisible(false);

    Alert.alert(
      "Changes Saved",
      "Do you want to reprint the case label?",
      [
        {
          text: "No",
          style: "cancel",
        },
        {
          text: "Yes",
          onPress: async () => {
            await handlePrintCaseLabel(updatedCase);
          },
        },
      ]
    );
  } catch (error) {
    console.error("[handleSaveCaseInfo] failed", error);
    Alert.alert("Error", "Could not save case changes.");
  } finally {
    setIsSavingCaseInfo(false);
  }
}
```

## 5. Add a print-case-label function

If you already have lab slip / label generation, reuse it.

```ts
async function handlePrintCaseLabel(caseRecord: any) {
  try {
    const html = buildCaseLabelHtml(caseRecord);

    await Print.printAsync({
      html,
      ...(Platform.OS === "ios" && savedPrinter?.url
        ? { printerUrl: savedPrinter.url }
        : {}),
    });
  } catch (error) {
    console.error("[handlePrintCaseLabel] failed", error);
    Alert.alert("Print Error", "Unable to print the updated case label.");
  }
}
```

# Backend rule

Your backend should also enforce this, not just the UI.

Patch route should check admin membership before saving:

```js
if (!membership || membership.role !== "admin") {
  throw forbidden("Only admin users can edit case info");
}
```

If owner should also be allowed:

```js
if (!membership || !["owner", "admin"].includes(membership.role)) {
  throw forbidden("Only admin users can edit case info");
}
```

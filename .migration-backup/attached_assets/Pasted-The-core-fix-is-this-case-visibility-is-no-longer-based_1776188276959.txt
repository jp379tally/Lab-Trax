The core fix is this: case visibility is no longer based on who currently shares the same practiceName. It is now based on a case-level lab snapshot taken when the case is created.
That changes the behavior the way you asked:
1.	A pending invite or pending join request does not pull that user’s cases into the lab.
2.	Cases created before someone joins a lab stay personal and do not carry over.
3.	Cases created while someone is in a lab are stamped to that lab and stay with the lab even if that user leaves later.
4.	Leaving a lab no longer drags those shared cases away.
5.	The home screen “Recent Cases” now sorts newest cases first and shows the actual case owner’s initials, so phone A and phone B render the same shared cases.
6.	Pull-to-refresh now refreshes the user’s lab membership context before refetching cases, so mobile refresh is more reliable.

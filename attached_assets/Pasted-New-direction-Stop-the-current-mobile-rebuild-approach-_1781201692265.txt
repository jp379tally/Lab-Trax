## Phase 1 — Mobile Viewer / Case Lookup Only

Remove old mobile functionality from the active app.

Phase 1 scope is intentionally basic:

1. Login / logout.
2. Load cases from the same database used by desktop/web.
3. Cases entered on desktop/web must appear on mobile.
4. Mobile must be able to search/look up cases.
5. User must be able to open a case, such as Erik Steffen, and see:

   * overview
   * restorations
   * notes
   * files
   * invoice
   * history
6. Read-only is acceptable for Phase 1.
7. No AI Reader in Phase 1.
8. No mobile case creation in Phase 1.
9. No mobile invoicing in Phase 1.
10. No mobile photo upload in Phase 1.
11. No barcode assignment in Phase 1.
12. No location changes in Phase 1.

The purpose of Phase 1 is to prove that mobile can accurately display the same desktop/web case data using the same database and workflows.

Phase 1 acceptance:

* User can log in.
* User can log out.
* Desktop-created cases appear on mobile.
* Erik Steffen’s case appears on mobile.
* Opening a case shows the same overview/restorations/notes/files/invoice/history as desktop/web.
* No local-only mobile case data.
* No `/api/legacy/cases`.
* No mobile-only invoice reconstruction.
* No mobile-only case history.
* No mobile-only attachments.

## Phase 2 — AI Reader + Case Intake + Pan Barcode

Only after Phase 1 passes, add mobile creation/intake.

Phase 2 scope:

1. AI Reader prescription capture.
2. Automatic prescription detection similar to OneDrive.
3. Toggle automatic shutter on/off.
4. Manual capture still available.
5. AI reads prescription.
6. Mobile follows the same case creation workflow as desktop/web.
7. Unknown provider/practice workflow must match desktop/web.
8. Case is created in the same database as desktop/web.
9. Invoice is created using the same desktop/web invoice workflow.
10. User assigns physical case pan using barcode scan.
11. Barcode is tied to the canonical case.
12. Barcode can then be used to locate the case throughout the lab.
13. User can hard-press/long-press case to locate it.
14. After barcode assignment, prompt:
    “Print case label?”
15. Printed case label uses the same format/data as desktop/web.
16. At the end of Phase 2:

* case is entered
* invoice is created
* pan barcode is assigned
* label print prompt has happened

Do not add other features before Phase 2 is approved.

# Regression: AI Reader / Drop Zone Fails With "An Unexpected Error Occurred"

## Summary

The Dashboard AI Reader / Drop Zone workflow appears to be broken.

Previously, users could drag a case document into the Drop Zone and the AI reader would process it and begin case creation.

Now, dropping a document into the Drop Zone immediately results in:

```text
An unexpected error occurred.
```

No case is created and no useful error information is presented to the user.

## Severity

High

This breaks one of the primary workflow accelerators in LabTrax and appears to be a regression from previously working behavior.

## Steps to Reproduce

1. Navigate to Dashboard.
2. Drag a case document into the Drop Zone.
3. Wait for AI processing to begin.

## Expected Result

* File uploads successfully.
* AI reader processes the document.
* Extracted case information is displayed.
* User proceeds through case creation workflow.

## Actual Result

* Red error panel appears in Drop Zone.
* Message shown:

```text
An unexpected error occurred.
```

* No additional details are provided.
* No case is created.

## Screenshot

Attached screenshot shows the Dashboard Drop Zone displaying:

```text
An unexpected error occurred.
```

while the remainder of the Dashboard continues functioning normally.

## Why This Appears To Be A Regression

The AI reader workflow was functioning previously.

The same workflow now fails before any visible processing occurs, suggesting one of the following:

### Frontend Issues

* Drop Zone upload handler failure
* Missing API route
* Bad request payload
* Failed upload initialization
* React state/runtime error

### Backend Issues

* AI extraction endpoint failure
* Authentication/session validation issue
* Storage upload failure
* OpenAI/AI provider integration error
* Queue worker failure
* OCR service failure

### Infrastructure Issues

* Missing environment variable
* Expired API key
* Storage bucket permission issue
* Background worker not running
* Service deployment regression

## UX Improvement

Current error:

```text
An unexpected error occurred.
```

is not actionable.

Replace with specific messages such as:

* Unable to upload document
* AI extraction service unavailable
* OCR processing failed
* Unsupported file type
* File exceeds upload limit
* Authentication expired

and log the actual error server-side.

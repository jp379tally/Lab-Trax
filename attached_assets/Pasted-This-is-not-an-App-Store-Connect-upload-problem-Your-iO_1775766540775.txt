This is not an App Store Connect upload problem. Your iOS archive is failing before upload completes. The key failure is in the React Native bundling step, where your app is trying to include Node/server-only modules in the iOS bundle.

The smoking gun is this part of your log:

Bundle React Native code and images
var fs = yield import("fs")
http = yield import("http")
https = yield import("https")
url = yield import("url")
Invalid expression encountered
ARCHIVE FAILED
PhaseScriptExecution Bundle React Native code and images

What that means:

Your mobile app code is pulling in something that depends on Node APIs like:

fs
http
https
url

Those are fine on a server, but they cannot be bundled into a native iPhone app. So Replit/Fastlane gets all the way to Xcode archive, then crashes when Metro/Hermes tries to bundle JavaScript for iOS.

Most likely causes:

You accidentally imported a server-side file into the app.
A shared utility file is being used by both server and app, but it imports Node stuff.
A package you added is browser/server oriented and not React Native compatible.
There is code using dynamic imports like import("fs") or a library that does that internally.

What to check in your project right now:

Search your app code for:

fs
http
https
url
import('fs')
import("fs")
require('fs')
require("fs")

Also search for imports from files like:

server/
server_dist/
api/
lib/
utils/
shared/

Especially look for cases like:

import { something } from "../shared/whatever";

If that shared file imports Node modules anywhere, the iOS build can fail even if you are not directly using the server-only function.

What usually needs to be changed:

If you have something like this in app code:

import { myFunction } from "../server/myServerFile";

that must be removed. The app should call your backend through HTTP/fetch, not import backend code directly.

Bad:

import { saveFile } from "../server/files";

Good:

const res = await fetch(`${API_URL}/save-file`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(payload),
});

If you have a shared file, split it like this:

shared/types.ts → okay for both app and server
server/fileHelpers.ts → server only
app/api/client.ts → app only

Safe shared files should contain only:

types
constants
plain validation
pure formatting helpers

They should not contain:

fs
path
http
https
url
server SDK setup
Express code
Node-only libraries

Very likely pattern in your case:

Because your earlier Replit issues mentioned server imports and shared schema confusion, I suspect the mobile app bundle is touching code that was originally meant for the backend. This new log strongly supports that.

This is a bundling/runtime compatibility issue for React Native/Hermes, not a signing/upload problem.

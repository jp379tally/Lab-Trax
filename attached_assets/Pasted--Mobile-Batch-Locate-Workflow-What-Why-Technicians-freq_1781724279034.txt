# Mobile Batch Locate Workflow

## What & Why

Technicians frequently move entire batches of case pans through production stages.

The current locate workflow is case-by-case and too slow for production.

Add a dedicated **Batch Locate** workflow to the mobile dashboard that allows a technician to continuously scan multiple case-pan barcodes and then move the entire batch to a new location in one action.

This is a mobile-only productivity feature that still uses the same canonical location system as desktop.

---

## Done Looks Like

A new **Batch Locate** button exists on the mobile dashboard.

Tapping it opens a continuous barcode scanning screen.

The scanner remains active after each successful scan.

As barcodes are scanned, the screen shows:

* Total count scanned
* Barcode number
* Patient name
* Case number (optional but recommended)

The same barcode cannot be counted twice.

When scanning is complete:

* User taps Continue
* User selects destination location/station
* All scanned cases are moved together
* Location updates immediately appear on desktop and mobile

## Duplicate Protection

Critical requirement:

Once a barcode is successfully scanned:

* ignore future reads of that barcode
* do not increment count
* do not create duplicate rows

Instead:

Show small notification:

Already scanned

or

Duplicate barcode ignored

Use an in-memory Set for the active scan session.

---

## Barcode Lookup

Each scan should:

1. Read barcode
2. Lookup canonical case
3. Retrieve:

   * patient name
   * case number
   * current location
4. Add result to scan list

No local-only lookup.

Must use canonical API.

---

## Invalid Barcode Handling

If barcode not found:

Show:

Barcode not found

Do not add to batch.

Continue scanning.

Scanner remains active.

* single batch operation if supported

---

## Case History

Each affected case must receive the same location history event generated by desktop locate workflows.

History should show:

Moved from Milling → Post Mill

or equivalent existing event structure.

---

## Error Handling

If some cases fail:

Show:

15 updated
2 failed

List failures.

Allow retry.

Do not roll back successful updates.

---

## Optional Nice-To-Have

Display current location beside each scan result.

Example:

00124 — John Smith
Current: Milling

This gives technicians a chance to catch mistakes before moving the batch.

---

## Regression Requirements

Must not break:

* single-case locate
* barcode search
* barcode assignment
* mobile case detail
* desktop locate workflow
* case history

## Acceptance

Accepted when:

1. User taps Batch Locate.
2. User scans multiple case-pan barcodes continuously.
3. Scanner never double-counts the same barcode.
4. Screen shows:

   * total count
   * barcode numbers
   * patient names
5. User selects destination location.
6. All cases move together.
7. Desktop reflects changes immediately.
8. History is recorded correctly.
9. No regression to existing barcode workflows.

"Mobile Batch Locate Workflow"

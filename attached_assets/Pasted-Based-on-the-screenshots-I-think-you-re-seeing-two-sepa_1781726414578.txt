Based on the screenshots, I think you're seeing **two separate permission-model problems**.

### Problem 1: You can't manage roles properly inside SDR1

In the SDR1 dialog, I can see:

* SDR1
* Your role = Owner
* Members:

  * JOHN PHILLIPS (Owner)
  * ForrestE (User)
  * ADLTEST (User)

You currently only have **Remove** buttons. There is no:

* Promote to Admin
* Demote from Admin
* Change role
* Suspend user
* Reactivate user
* Transfer ownership
* Permission matrix

For a lab owner, that's incomplete.

### What should exist

For every member of SDR1:

| Permission           | Owner | Admin    |
| -------------------- | ----- | -------- |
| Invite users         | ✓     | ✓        |
| Remove users         | ✓     | ✓        |
| Change user role     | ✓     | ✓        |
| Manage providers     | ✓     | ✓        |
| Manage cases         | ✓     | ✓        |
| Manage invoices      | ✓     | Optional |
| Manage subscriptions | ✓     | No       |
| Transfer ownership   | ✓     | No       |
| Delete organization  | ✓     | No       |

Each member row should have:

```
[Role Dropdown]
Owner | Admin | Billing | Technician | User | Read Only
```

and changing the dropdown should immediately update permissions.

# Problem 2: You appear as Owner of a bunch of dental practices

This is the bigger bug.

In the Organizations screen I can see entries like:

* CWD Dental Group-Kerry Forest Pkwy
* Brittney K Craig DMD
* Family Dentistry at SouthWood
* Heartland Dental
* etc.

and every one says:

```
Owner
```

That makes no sense if SDR1 is your only organization.

### What is probably happening

LabTrax currently appears to be treating provider organizations as if they are organizations you belong to.

There should be a distinction between:

### Organizations I belong to

Examples:

* SDR1

These are organizations where:

* I have membership
* I have a role
* I can administer users

### Organizations referenced by cases

Examples:

* Brittney K Craig DMD
* CWD Dental Group
* Family Dentistry at SouthWood

These are provider records.

You should NOT automatically become:

* Owner
* Admin
* Member

of every provider record simply because a case references that provider.

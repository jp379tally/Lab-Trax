### Debug Analysis

The highest-confidence issue is the **due date reverting** bug. That usually happens when a mutation succeeds but old cached data wins afterward, or when the frontend sends one date format and the backend returns another. Web is specifically affected, so Replit should inspect web case edit state, API response shape, and query invalidation first.

The second major pattern is **cross-platform drift**. Mobile lab slips not matching web/desktop and mobile landing on Cases instead of Dashboard both point to separate platform implementations instead of shared behavior. The best fix is not another mobile-only patch; it is shared templates and shared route defaults wherever possible.

The delete case and add practice regressions likely live at the boundary between frontend mutation, backend route, DB operation, and cache invalidation. Replit should avoid changing only the button handlers. The fix needs to cover API contract, persistence, and post-mutation UI refresh.

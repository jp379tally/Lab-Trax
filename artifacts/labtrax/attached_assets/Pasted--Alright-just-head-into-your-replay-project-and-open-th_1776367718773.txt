4. Merge instead of overwrite
A lot of these regressions happen because one code path does:
setActivityEntries(serverEvents);

and wipes out locally-added items, or a stale fetch wipes out newer ones.
Use a merge helper:
function mergeHistoryEvents(
  existing: CaseHistoryEvent[],
  incoming: CaseHistoryEvent[]
): CaseHistoryEvent[] {
  const map = new Map<string, CaseHistoryEvent>();

  for (const item of existing) {
    map.set(item.id, item);
  }

  for (const item of incoming) {
    map.set(item.id, item);
  }

  return Array.from(map.values()).sort(
    (a, b) => new Date(a.createdAt).getTime() - new Date(b.createdAt).getTime()
  );
}

Then: setActivityEntries((prev) => mergeHistoryEvents(prev, fetchedEvents));

Never blindly replace with stale data unless you know it is authoritative and complete.

import { Platform, Alert } from "react-native";
import * as Print from "expo-print";

export async function choosePrinter() {
  if (Platform.OS !== "ios") return null;
